# 🐞 Bug Reports Directory

## 📌 Overview

This folder contains formal bug documentation identified during validation of the Benefits Dashboard application.

The reports in this directory were created to capture reproducible issues, document expected versus actual behavior, and preserve evidence for QA communication and root-cause analysis.

---

## 📁 Contents

### `BUG_TEMPLATE.md`

Reusable template for documenting defects in a structured and consistent way.

### `BUG-001_Employees_Table_Loads_Empty_After_Automated_Navigation.md`

Documents the issue where the employees table may render empty after automated navigation, even when employee records exist.

### `BUG-002_Employee_Modal_Remains_Open_After_Submission.md`

Documents the issue where the employee modal may remain open after submission, even though the employee record is successfully persisted.

---

## 🎯 Purpose

This folder exists to provide:

- Structured defect documentation
- Reproducible QA findings
- Stakeholder-ready communication
- Evidence-backed issue tracking
- Traceability between UI findings and API validation

---

## 🏁 Final Note

These reports represent the formal defect documentation layer of the testing challenge and help demonstrate professional QA communication and issue analysis.
